# 提示词修改与管理指南

本文档旨在帮助开发者和提示词工程师快速定位、修改和管理 Meon-Clone 项目中的 AI 提示词（Prompts）

## 📂 核心文件位置

所有 AI 相关的底层提示词逻辑已集中管理在 `services/ai/` 目录下：

| 文件路径 | 用途描述 |
| :--- | :--- |
| **`services/ai/prompts.ts`** | **[核心]** 动态提示词构建逻辑。包含所有根据用户输入（如剧本、角色信息）生成最终 Prompt 的函数。 |
| **`services/ai/promptConstants.ts`** | **[常量]** 静态提示词片段。包含视觉风格定义（Visual Styles）、负面提示词（Negative Prompts）等。 |

---

## 🛠️ 如何修改提示词

### 1. 修改视觉风格与负面提示词 (`promptConstants.ts`)

如果您需要调整某种风格的画风描述，或添加通用的负面词，请编辑 `services/ai/promptConstants.ts`。

*   **视觉风格 (VISUAL_STYLE_PROMPTS)**: 定义了 `live-action`, `anime`, `cyberpunk` 等风格的具体英文描述。
*   **中文名称 (VISUAL_STYLE_PROMPTS_CN)**: 定义了 UI 上显示的风格名称。
*   **负面提示词 (NEGATIVE_PROMPTS)**: 定义了不同风格对应的通用负面词（如 `bad anatomy`, `text`, `watermark`）。

**示例：修改 "动漫风格" 的描述**

```typescript
// services/ai/promptConstants.ts

export const VISUAL_STYLE_PROMPTS: { [key: string]: string } = {
  // ...
  'anime': 'Japanese anime style, cel-shaded, vibrant colors, ... (在此处修改详细描述)',
  // ...
};
```

### 2. 修改业务逻辑提示词 (`prompts.ts`)

如果您需要调整 AI 的具体指令（例如：如何解析剧本、如何设计角色、如何生成分镜），请编辑 `services/ai/prompts.ts`。

该文件按功能模块划分为多个区域：

#### A. 剧本服务 (Script Service)
*   **`buildScriptParsingPrompt`**: 用于将粗略文本解析为结构化剧本（JSON）。
*   **`buildScriptContinuationPrompt`**: 用于剧本续写。
*   **`buildScriptRewritePrompt`**: 用于剧本润色/改写。
*   **`buildShotListGenerationPrompt`**: 用于将场次拆分为具体的分镜列表。

#### B. 视觉服务 (Visual Service)
*   **`buildArtDirectionPrompt`**: 生成全局美术指导文档（Art Direction Brief）。
*   **`buildCharacterPrompt`**: 生成单个角色的视觉提示词。
*   **`buildBatchCharacterPrompt`**: 批量生成所有角色的提示词。
*   **`buildScenePrompt`**: 生成场景环境的视觉提示词。
*   **`buildOutfitVariationPrompt`**: 角色换装（保持人脸一致，更换服装）。
*   **`buildConsistencyPrompt`**: 角色一致性控制（用于生成分镜图时保持角色长相）。
*   **`buildTurnaroundPanelPrompt`**: 生成角色九宫格视图的单格描述。

#### C. 分镜与视频服务 (Shot & Video Service)
*   **`buildOptimizeBothKeyframesPrompt`**: 同时优化分镜的起始帧和结束帧描述。
*   **`buildDetailedActionSuggestionPrompt`**: 根据首尾帧生成中间动作描述。
*   **`buildDetailedShotSplitPrompt`**: 将一个长镜头拆分为多个细分镜头。
*   **`buildNineGridPanelsPrompt`**: 生成九宫格分镜的 9 个视角描述。

## 📝 修改示例

### 场景：我想让 AI 生成的角色描述更详细

1.  打开 `services/ai/prompts.ts`。
2.  搜索 `buildCharacterPrompt` 函数。
3.  在返回的模板字符串中添加具体要求。

```typescript
// services/ai/prompts.ts

export const buildCharacterPrompt = (...): string => {
  return `Generate a detailed visual prompt for this character:
  Name: ${char.name}
  ...
  
  Requirements:
  - Detailed description of facial features...
  - [新增] Describe the texture of the skin and fabric in detail. (在此添加新要求)
  - [新增] Ensure the lighting reflects the character's mood.
  ...
  `;
};
```

### 场景：我想调整剧本解析的 JSON 格式

1.  打开 `services/ai/prompts.ts`。
2.  搜索 `buildScriptParsingPrompt` 函数。
3.  修改 `Requirements` 部分和 JSON 结构示例。

**注意：** 如果修改了 JSON 结构，请务必同步修改 `services/ai/scriptService.ts` 中解析该 JSON 的 TypeScript 接口定义，否则代码会报错。

---

## ⚠️ 注意事项

1.  **JSON 格式**: 很多提示词要求 AI 返回 JSON 格式。修改时请保留 `Return a valid JSON object...` 等关键指令，确保 AI 输出可被代码解析。
2.  **变量插值**: 提示词中使用 `${variable}` 插入动态数据。请小心不要删除这些关键变量（如 `${visualStyle}`, `${char.name}`）。
3.  **语言控制**: 部分提示词明确要求输出英文（用于绘图）或中文（用于剧本）。修改时请留意 `Output in English` 或 `Language: ${language}` 等指令。
4.  **Prompt 长度**: 尽量保持提示词精简。过长的提示词可能导致 AI 遗忘前面的指令或增加 API 成本。
