# 剧集大项目管理功能 - 实施计划

## 目标
在现有「项目」体系之上，新增「剧集大项目管理」能力，使用户可以在同一个项目下创建/管理季与集，并进入单集编辑；同时提供旧项目迁移到新版结构的选择与自动迁移能力。新增 UI 需适配深浅色主题。

## 用户流程（对齐需求）
### 新建项目（新版默认）
1. Dashboard 点击「新建项目」
2. 直接进入「大项目管理」页面
3. 新建剧集（季/Season）→ 输入名称并确认
4. 点击「创建第一集」或「+」创建集（Episode）
5. 点击集数进入该集的编辑页面（沿用现有 Script/Assets/Director/Export/Prompts 工作流）

### 打开旧项目（迁移选择）
1. 用户打开旧项目（当前结构）
2. 弹窗询问：是否迁移到新版大项目管理？
   - 选择「是」：自动创建「第一季-第一集」，并把迁移前编辑器内容作为第一集内容
   - 选择「留在旧版」：保持原逻辑与数据结构继续使用

### 数据共享逻辑
- 每一集的数据彼此独立（剧本/解析结果/镜头/生成记录等），但同一项目下的角色库/场景库/道具库可共用并逐集累积。
- 后续集新增的角色/场景/道具应能在更后续集被选择使用。
- 当某集分镜/文本提到其他集已有角色时，自动推荐添加（推荐层面，不强制）。

---

## 现状调研结论（基于仓库）
- 当前没有真实路由：新建项目由 Dashboard 触发，App 通过 `project === null` 切换视图。
- `ProjectState` 为单一项目单一工作流：`rawScript/scriptData/shots/stage` 等都在项目根上。
- 暗/亮主题通过 `data-theme` + CSS Variables 实现，组件用 Tailwind arbitrary values 引用变量。

---

## 方案总览（高层设计）
引入「项目格式版本」与「剧集结构」：
- 新建项目默认使用新版（v2）：进入「大项目管理」Stage。
- 旧项目保持 v1；打开 v1 时弹迁移选择。
- v2 项目在项目根上维护“共享资产库”（角色/场景/道具），每一集维护自己的创作数据（剧本/解析/镜头/生成记录/当前 stage 等）。
- 进入某集编辑时，UI 仍复用现有各 Stage 组件，但其数据来源改为“当前集 + 共享资产库”的组合视图。

---

## 数据模型改造（Types & Storage）
### 1) 新增类型（建议）
- `ProjectFormatVersion = 'v1' | 'v2'`
- `SeriesSeason`：`id/title/createdAt/episodeIds(or episodes[])`
- `SeriesEpisode`：`id/title/createdAt/lastModified/status`
- `EpisodeState`：包含原 `ProjectState` 中“与单次创作相关”的字段子集：
  - `stage`（script/assets/director/export/prompts）
  - `rawScript/targetDuration/language/visualStyle/shotGenerationModel`
  - `scriptData/shots/isParsingScript/renderLogs`
- `SharedLibrary`：`characters/scenes/props`（沿用现有类型）
- `ProjectState` 扩展字段（向后兼容）：
  - `formatVersion?: ProjectFormatVersion`（v1 缺省）
  - `series?: { seasons: SeriesSeason[]; episodes: Record<string, EpisodeState>; activeEpisodeId?: string; expandedSeasonIds?: string[] }`
  - `sharedLibrary?: SharedLibrary`
  - `migrationPreference?: 'ask' | 'stay_legacy' | 'migrated'`（用于避免每次打开都弹窗）

### 2) IndexedDB / Supabase 同步影响
- IndexedDB：当前直接存 `ProjectState` 全量对象；新增字段保持可序列化即可，旧数据读取不受影响。
- Supabase：若当前开启项目同步，需要更新 `syncService` 对 `ProjectState` 序列化/反序列化映射，确保 v2 字段不丢失（或作为 JSON blob 原样存储）。
- 迁移应确保不写入敏感信息，且不会破坏旧项目在“留在旧版”情况下的可用性。

## UI/交互实现
### 1) 新增 Stage：大项目管理
新增一个工作区 Stage（例如 `stage: 'series'` 或 `'project'`）：
- 在 App 的 `project.stage` switch 中渲染新组件（SeriesManager）。
- Sidebar 增加入口（仅 v2 可见或在 v2 默认置顶）。

### 2) 大项目管理页面（参考文件对齐）
实现参考文件中的布局与交互：
- 统计卡：剧集数、总集数
- 引导区：3 步新手引导
- 剧集管理区：
  - 新建剧集（命名表单、回车创建、创建/取消）
  - 剧集展开/折叠
  - 新增集（空态「创建第一集」、头部 +、底部「添加新集」）
  - 删除剧集/删除单集（二次确认弹窗）
  - 点击集进入该集编辑：设置 `activeEpisodeId` 并切到该集上次 `stage`（若无则从 script 开始）

### 3) 迁移弹窗（旧项目打开时）
触发时机：App 在 `handleOpenProject` 后检测该项目是否为 v1 且 `migrationPreference === 'ask'`（或缺省）。
弹窗选项：
- 「是，迁移到新版」：执行 v1→v2 迁移并进入大项目管理或直接进入第一集编辑（以需求为准，默认先进入大项目管理并高亮第一季第一集）。
- 「留在旧版」：写入 `migrationPreference='stay_legacy'`，后续不再弹；继续原 stage 渲染。

## 共享资产库与“推荐添加”逻辑
### 1) 共享库累积
- v2 项目根上维护 `sharedLibrary.characters/scenes/props`。
- 每次在某集解析剧本产生新的角色/场景/道具时：
  - 用“名称/关键字段”做去重（角色：name；场景：location+time；道具：name）
  - 新条目写入 sharedLibrary，同时把 episode 的 `scriptData` 引用指向 sharedLibrary 中的对象（避免复制导致分叉）

### 2) 每集独立但可复用
- 每集维护 `episode.usedCharacterIds/usedSceneIds/usedPropIds`（如需要更强约束），用于：
  - 资产页默认只展示“本集已使用”的条目，同时提供“从共享库添加”入口
  - 分镜页选角色/场景/道具时默认从“本集已使用”中选，但可一键从共享库添加

### 3) “自动推荐添加角色”
- 在导演页/分镜编辑中，当用户输入 actionSummary/dialogue 时：
  - 用简单的字符串匹配（角色名出现在文本中）从 sharedLibrary 找到候选角色
  - 若候选不在 `usedCharacterIds`，在 UI 显示「推荐添加」chips（点击即加入本集 usedCharacterIds）
- 该能力先做到“推荐提示”，不改变现有生成流程与数据结构。

## 具体实施步骤（按落地顺序）
1. **类型与数据结构**
   - 扩展 `types.ts`：加入 v2 所需类型、扩展 `ProjectState`，并确保不破坏现有编译。
2. **新建项目改造**
   - 新增 `createNewProjectStateV2()`：创建空 sharedLibrary + 空 seasons/episodes；默认 `stage` 为大项目管理。
   - Dashboard 新建项目入口改为创建 v2 项目并打开。
3. **新增大项目管理 Stage**
   - 新建组件 `components/SeriesManager`（名称以仓库风格为准），复刻参考 UI 与交互。
   - 在 App 的 `stage` switch 与 Sidebar 中接入。
4. **单集编辑接入（复用现有工作流）**
   - 在 App 层引入 “当前集视图” 的选择器：将 `project + activeEpisodeId` 映射为现有 Stage 组件所需 props。
   - 调整 `updateProject` 以支持对 episode 的局部更新（例如 `updateEpisode(episodeId, updates)`），并保持自动保存逻辑不变。
5. **共享资产库接入各 Stage**
   - 调整 StageScript/StageAssets/StageDirector/StagePrompts 等读取 `scriptData.characters/scenes/props` 的逻辑：
     - v1：保持原样
     - v2：使用 `sharedLibrary` 作为权威库；episode 仅保留 storyParagraphs/title 等“剧情结构”
6. **迁移弹窗与迁移实现**
   - 在打开 v1 项目时弹窗；实现 v1→v2 一键迁移（第一季/第一集填充旧数据）。
   - 支持“留在旧版”并记忆。
7. **深浅色适配与样式对齐**
   - 所有新增 UI 仅使用现有 CSS 变量令牌（`var(--bg-*)/var(--text-*)/var(--border-*)`），不硬编码颜色。
8. **验证与回归**
   - TypeScript 编译通过。
   - 新建项目→剧集管理→创建季/集→进入编辑→各 Stage 正常工作→退出项目后重新打开仍保持结构与数据。
   - 旧项目：选择迁移/留旧都可正常使用，且选择会被记住。
   - 深/浅色主题下页面可读性、边框/hover/disabled 状态一致。
9. **交付简洁报告**
   - 输出变更点清单、核心数据结构说明、用户操作验证路径、已知限制/后续建议。

---

## 风险与控制
- **改动面较大（多 Stage 组件引用 scriptData）**：通过“选择器 + 兼容分支（v1/v2）”逐步接入，避免一次性重写。
- **去重与引用一致性**：sharedLibrary 以 id 为主，按 name/关键字段做合并策略；避免同名多 id 导致推荐与选择混乱。
- **同步服务兼容**：若启用 Supabase，同步层必须按 JSON 结构完整保存 v2 字段，避免云端覆盖导致丢失。

## 非目标（本轮不做或延后）
- 多人协作与权限体系
- 复杂 NLP 角色识别（本轮用轻量字符串匹配实现“推荐”）
- 跨项目共享资产库规则重设（仍沿用现有 assetLibrary 逻辑）
